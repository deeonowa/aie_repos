# Итоговый проект по курсу «Инженерия Искусственного Интеллекта»

В этой папке находится итоговый мини-проект по курсу.  
Проект демонстрирует применение методов и инструментов инженерии ИИ: работу с данными, модели, пайплайны, сервис, эксперименты и воспроизводимость.

Подробный отчёт (постановка задачи, эксперименты, обоснование модели) — в файле `report.md`.

---

## 1. Паспорт проекта

**Название проекта:** CVE Severity Triage API — сервис первичной приоритизации уязвимостей по тексту описания CVE.

**Автор:** `Зуенко Андрей Андреевич`

**Группа:** `Группа БИСО-01-23`

**Контакт:** `@deeonowa`

**Краткое описание (2–4 предложения):**

Проект посвящён построению ML-сервиса для аналитика vulnerability management: по английскому тексту описания CVE из NVD модель относит уязвимость к одному из трёх классов серьёзности — **low / medium / high** (пороги CVSS v3 base: 4.0 и 7.0). Используются открытые данные NVD **2024 и 2025** (~78k записей), TF-IDF-признаки и классические модели (Dummy, Logistic Regression, SGD, Random Forest). Результат — REST API на FastAPI с эндпоинтами `/health` и `/predict`, ноутбуки с EDA и экспериментами, артефакты и графики в `artifacts/figures/`.

---

## 2. Структура проекта

Проект организован в следующей структуре:

- `requirements.txt` – зависимости проекта (библиотеки Python, необходимые для запуска).
- `report.md` – отчёт по проекту (постановка задачи, данные, эксперименты, результаты).
- `self-checklist.md` – чеклист самопроверки проекта перед сдачей.
- `pytest.ini` – настройки запуска тестов.
- `notebooks/` – экспериментальные ноутбуки:
  - `01_eda.ipynb` – EDA данных NVD;
  - `02_experiments.ipynb` – сравнение моделей на подвыборке.
- `src/` – основной код проекта:
  - `data/` – загрузка и подготовка данных NVD;
  - `features/` – предобработка текста описания CVE;
  - `models/` – обучение, инференс, пайплайны и метрики;
  - `service/` – FastAPI-сервис (`/health`, `/predict`);
  - `utils/` – загрузка конфигурации;
  - `train.py` – точка входа обучения (`python -m src.train`).
- `data/` – данные для обучения и демонстрации:
  - `sample_cve.csv` – небольшая выборка для быстрого старта;
  - `processed/` – полный CSV после `python -m src.data`;
  - `raw/` – сырые архивы NVD (при загрузке данных).
- `configs/` – конфигурационные файлы:
  - `config.yaml` – параметры данных, модели и сервиса;
  - `.env.example` – шаблон переменных окружения (порт, путь к модели, уровень логов).
- `tests/` – тесты (юнит-тесты, smoke- и sanity-проверки API).
- `artifacts/` – сохранённые модели, метрики и графики экспериментов:
  - `cve_model.joblib` – модель для API;
  - `metrics_all.json`, `best_model.json` – метрики экспериментов и финальной модели;
  - `figures/` – графики EDA и сравнения моделей.
- `Dockerfile`, `docker-compose.yml` – контейнеризация и локальное развёртывание сервиса.

## 3. Требования и установка

### 3.1. Требования

- Python >= 3.10
- Docker (для запуска контейнера)

### 3.2. Установка окружения

Для локального запуска (API, ноутбуки, тесты, обучение). Команда `python` должна быть версии **3.10 или выше** (проверка: `python --version`).

```bash
cd project

python -m venv .venv
source .venv/bin/activate   # Windows: .venv\Scripts\activate

pip install --upgrade pip
pip install -r requirements.txt
```

---

## 4. Как запустить проект

В репозитории уже есть обученная модель `artifacts/cve_model.joblib`. Для запуска API переобучение не требуется.

### 4.1. Запуск через Docker

1. Сборка образа:

```bash
cd project
docker build -t cve-severity-triage-api .
```

2. Запуск контейнера:

```bash
docker run -p 8000:8000 cve-severity-triage-api
```

Первая сборка может занять несколько минут (установка зависимостей в образе). Повторные сборки быстрее, если не менялся `requirements.txt`.

### 4.2. Локальный запуск

```bash
cd project
source .venv/bin/activate 
python -m src.service
```

- Сервис доступен по адресу: http://localhost:8000
- Эндпоинты:
  - `/docs` — документация API (Swagger UI)
  - `/health` — статус сервиса и загрузка модели
  - `/predict` (POST) — класс серьёзности (`low` / `medium` / `high`) по тексту описания CVE

Все эндпоинты также доступны через http://localhost:8000/docs.

## 5. Данные

- **Источник:** открытые feed NVD 2024 и 2025 ([FKIE CAD nvd-json-data-feeds](https://github.com/fkie-cad/nvd-json-data-feeds))
- **Метки:** `low` (CVSS < 4.0), `medium` (4.0–7.0), `high` (CVSS ≥ 7.0)

**Основные файлы:**

- `data/sample_cve.csv` — демо-выборка (~300 записей)
- `data/processed/cve_dataset.csv` — полный датасет (~78k), создаётся командой `python -m src.data`
- `data/raw/` — сырые архивы NVD
- `artifacts/cve_model.joblib` — модель для API
- `artifacts/best_model.json`, `artifacts/metrics_all.json` — метрики обучения и экспериментов
- `artifacts/figures/` — графики EDA и сравнения моделей

**Переобучение модели** (если нужно воспроизвести пайплайн с нуля):

```bash
cd project
source .venv/bin/activate

python -m src.data
python -m src.train
```

Ноутбуки:

- `notebooks/01_eda.ipynb` — EDA, графики в `artifacts/figures/`
- `notebooks/02_experiments.ipynb` — сравнение моделей на подвыборке, `artifacts/metrics_all.json`

## 8. Ограничения и дальнейшая работа

**Текущие ограничения:**

- модель обучена только на **английских** описаниях NVD;
- метки — **generic CVSS**, без учёта активов организации;
- класс **low** редкий (~2% выборки) — ниже recall по low;
- в API одна модель (**LogReg**);
- данные — **2024–2025**, без более ранних лет;
- авторизация в API не реализована;

**Направления развития:**

- fine-tuning трансформера для текста CVE;
- периодическое обновление feed и переобучение;
- метрики наблюдаемости (Prometheus), CI-сборка образа после train.
